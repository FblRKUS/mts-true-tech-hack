# Фича: Суд Нейросетей (Model Court)

## Цель

Запускать несколько моделей на одну задачу, а затем в отдельном шаге агрегировать и выбирать лучший итоговый ответ по заданным критериям.

## Почему это важно для оценки

- Может усилить качество ответов на сложных запросах.
- Показывает продвинутую автоматизацию.
- Дает сильный демо-эффект при честной прозрачной реализации.

## Как фича связана с общим сценарием

Пользователь включает режим "Суд" -> система запускает N моделей -> судья анализирует анонимизированные ответы -> пользователь получает объединенный итог с объяснением выбора.

## Где работать

- Backend:
  - Оркестратор параллельных вызовов моделей.
  - Анонимизация и нормализация промежуточных ответов.
  - Судья-агрегатор с критериями выбора.
- Frontend:
  - Переключатель режима "Суд".
  - Представление результатов: итог + краткое объяснение.
  - Управление лимитами (время/стоимость/число моделей).
- Наблюдаемость:
  - Логи, задержка, стоимость и успешность режима.

## План действий

- [ ] Определить условия, когда режим уместен.
- [ ] Определить критерии оценки ответов и формат итогового синтеза.
- [ ] Реализовать параллельный запуск моделей и агрегатор.
- [ ] Ограничить режим по времени и бюджету.
- [ ] Добавить понятное отображение итогового решения.
- [ ] Протестировать на задачах с высокой неоднозначностью.

## Риски и зависимости

- Риск: высокая задержка и стоимость.
- Риск: иллюзия качества без реального выигрыша.
- Зависимость: стабильный автороутинг и оркестратор.

## Критерий готовности для демо

- По одной задаче видно сравнение и итоговый выбор.
- Пользователь понимает, почему выбран именно этот итог.
- Режим укладывается в адекватное время ответа.
