# Фича: Превью PDF и Архивов Без Скачивания

## Цель

Дать пользователю быстрый просмотр PDF и структуры архивов прямо в интерфейсе чата, чтобы снизить лишние скачивания и ускорить работу с файлами.

## Почему это важно для оценки

- Закрывает реальный пользовательский сценарий "работа с файлами в одном окне".
- Улучшает UX и демонстрирует зрелость мультимодального контура.
- Удобно показывается в демо как практическая польза.

## Как фича связана с общим сценарием

Пользователь прикладывает файл -> открывает превью -> смотрит содержимое/структуру -> отправляет уточняющий запрос агенту по этому файлу в том же диалоге.

## Где работать

- Backend:
  - Сервис извлечения превью PDF (страницы/текстовые фрагменты).
  - Сервис чтения структуры архивов (без полной распаковки пользователю).
  - API безопасной выдачи превью-данных.
- Frontend:
  - Компонент предпросмотра PDF.
  - Компонент дерева содержимого архива.
  - Интеграция предпросмотра в карточку вложения чата.

## План действий

- [ ] Определить минимальный функционал предпросмотра для PDF и архивов.
- [ ] Реализовать backend-контур получения превью-данных.
- [ ] Собрать UI просмотра без скачивания.
- [ ] Связать превью с текущим сообщением и контекстом чата.
- [ ] Проверить поведение на больших файлах и ошибках формата.

## Риски и зависимости

- Риск: тяжелые файлы влияют на время ответа.
- Риск: небезопасная обработка архивов.
- Зависимость: общий модуль вложений и файлового storage.

## Критерий готовности для демо

- PDF можно открыть и просматривать в интерфейсе.
- Архив показывает дерево файлов/папок без скачивания.
- Пользователь может сразу задать вопрос по увиденному содержимому.
