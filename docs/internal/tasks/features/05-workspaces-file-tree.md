# Фича: Пространства и Дерево Файлов (Codex-подобный Режим)

## Цель

Сделать рабочие пространства с понятной структурой папок/файлов, где пользователь и агент видят один и тот же контекст проекта.

## Почему это важно для оценки

- Повышает ценность продукта для рабочих сценариев (код, документы, артефакты).
- Укрепляет идею "единый интерфейс для всех задач".
- Может стать сильным дифференциатором при корректной реализации.

## Как фича связана с общим сценарием

Пользователь открывает пространство -> управляет файлами в дереве -> запускает задачу в чате -> агент работает с тем же набором файлов -> результаты сохраняются в это же пространство.

## Где работать

- Backend:
  - Модуль пространств, прав доступа и файловых операций.
  - Индексация содержимого файлов для работы агента.
  - Ограничение и аудит операций агента с файлами.
- Frontend:
  - UI дерева файлов/папок.
  - Действия с файлами: открыть, переименовать, переместить, удалить.
  - Связь текущего пространства с окном чата.
- Безопасность:
  - Политики sandbox для операций с файловой системой.

## План действий

- [ ] Определить сущности: пространство, директория, файл, контекст сессии.
- [ ] Определить допустимые операции агента и пользователя.
- [ ] Реализовать базовое дерево и навигацию.
- [ ] Связать пространство с контекстом чата и инструментами.
- [ ] Добавить аудируемость файловых действий.
- [ ] Подготовить сценарий "задача -> изменения в файлах -> итог в чате".

## Риски и зависимости

- Риск: большой объем работ, который может вытеснить обязательные фичи.
- Риск: безопасность файловых операций.
- Зависимость: зрелость модели инструментов и ролевая модель доступа.

## Критерий готовности для демо

- Пользователь в одном интерфейсе управляет файлами и общается с агентом.
- Агент видит выбранное пространство и работает только в его пределах.
- Изменения в файлах прозрачно отражаются в UI.
