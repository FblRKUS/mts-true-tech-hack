# Стратегия Хакатона

## Цель

Собрать цельный продукт на базе OpenWebUI, который закрывает повседневные задачи пользователя в едином чате и демонстрирует сильную автоматизацию, память и мультимодальность.

## Продуктовые принципы

- Один интерфейс для всех типов задач: текст, голос, изображения, файлы.
- Автоматический выбор модели по типу задачи и по характеристикам моделей (доменная специализация, качество, скорость, стоимость, длина контекста) + обязательный ручной выбор модели.
- Долговременная память используется в реальных сценариях и контролируется пользователем.
- Никаких незавершенных "пустышек" в интерфейсе: показываем только рабочее.
- Все фичи выглядят как единый сценарий, а не как набор отдельных модулей.

## Приоритеты реализации

1. Авто модель (авторежим маршрутизации).
2. Память чата.
3. Превью PDF и архивов.
4. Фото OCR.
5. Единая мультимодальная модель сообщений.
6. Автопилот.
7. Медиа галерея.
8. Суд нейросетей.
9. Пространства и дерево файлов.

## Как работаем командой (2 разработчика)

- Архитектура: сервисный монолит.
- Ветки: `<ник>/feature/<название-фичи>`.
- Один PR = одна фича или один логический этап фичи.
- Не смешиваем независимые фичи в одной ветке.
- По каждой фиче заранее фиксируем зоны ответственности (backend / frontend / интеграции).
- Сначала согласуем контракты API и форматы данных, затем параллелим реализацию.

## Правила по реализации фич

- Каждая фича описана в `docs/internal/tasks/features/*.md`.
- В карточке фичи есть цель, зоны работ и план действий.
- Если фича не готова к показу, она не должна быть активной в UI.
- Готовые фичи фиксируем в `docs/internal/описание задачи/доп материал/GPTHub шаблон фич.xlsx`.

## Обязательные артефакты на сдачу

- Репозиторий с рабочим кодом и инструкцией запуска.
- Однострочный запуск решения через Docker/Docker Compose.
- Архитектурная схема: user flow + контур сервисов/моделей.
- Заполненный шаблон фич (`.xlsx`).
- Демо-видео с реальной работой всех заявленных сценариев.
- Презентация с проблемой, решением, архитектурой и ценностью.

## Критерии готовности

- Обязательные фичи работают стабильно: авторежим, память, ручной выбор модели.
- Мультимодальные сценарии связаны между собой и проходят end-to-end.
- UX последовательный: от отправки сообщения до результата без разрывов.
- Документация, демо и презентация соответствуют фактически реализованному функционалу.
