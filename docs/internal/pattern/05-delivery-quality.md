# Доставка и качество

## Запуск

- Однострочный запуск через Docker Compose.
- Минимум сервисов: `app`, `db`, `frontend` (если отдельно), `migrations`.
- Healthchecks обязательны.
- Конфигурация через `.env` и `.env.example`.

## Стандарты кода

- Явная типизация в Python/TypeScript.
- Имена отражают бизнес-смысл.
- Комментарии только для нетривиальной логики.
- Единый форматтер/линтер в проекте.

## Тесты

- Unit: сервисы и репозитории.
- Integration: API + DB + ключевые сценарии.
- Минимум один E2E-поток для демо.
- Приоритет покрытия: автомодель, память, мультимодальность.
