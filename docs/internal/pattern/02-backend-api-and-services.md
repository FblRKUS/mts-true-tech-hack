# Backend: API, сервисы, репозитории

Слои: `Router -> Service -> Repository -> Model`.

## Router

- Только HTTP-контракт: валидация, auth, mapping ошибок.
- Для каждого endpoint фиксируются request/response schema и коды ошибок.
- Версия API: `/api/v1`.

## Service

- Центр бизнес-логики и оркестрации.
- Один публичный метод = один use-case.
- Владеет транзакцией (`commit/rollback`).
- Междоменные вызовы только через публичные сервисы домена.
- Для AI-фич: роутинг моделей, память, мультимодальность координируются в сервисах.
- Решения автомодели должны быть объяснимыми (причина выбора).

## Repository

- Только доступ к данным и запросы.
- Без бизнес-валидации.
- Без `commit`.
- Работает только с таблицами своего домена.

## API-качество

- Единый формат ошибок: `code/message/details`.
- Пагинация и фильтрация единообразны.
- OpenAPI-описания обязательны (`summary/description`).
