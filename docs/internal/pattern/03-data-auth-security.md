# Данные, аутентификация и безопасность

## БД и миграции

- PostgreSQL + Alembic.
- Каждая миграция: `upgrade` и `downgrade`.
- Время: timezone-aware timestamp.
- Технический аудит в таблицах: `created_at`, `updated_at`.
- Soft delete использовать только там, где это обосновано.
- Индексы и уникальные ограничения задавать явно.

## Аутентификация и права

- JWT/Bearer.
- Авторизация policy/resource-based (`action` над `resource`).
- Проверка прав выполняется до бизнес-операций.

## Безопасность

- Секреты только из env.
- Не доверять идентификаторам из непроверенного body.
- Не логировать секреты и чувствительные данные.
- Ошибки auth не раскрывают внутренние детали.
