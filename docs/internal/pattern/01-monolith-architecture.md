# Монолитная архитектура

Базовый стиль: модульный сервисный монолит.

## Принципы

- Один backend + один frontend + одна БД.
- Один API-вход: `/api/v1`.
- Домены: `routing`, `memory`, `messages`, `files`, `media`, `auth`, `workspace`.
- Между доменами взаимодействие только через сервисные интерфейсы.
- Владение данными закреплено за доменом-владельцем.

## Антипаттерны

- Преждевременный переход к микросервисам.
- Бизнес-логика в роутерах.
- Прямой доступ к чужим таблицам/репозиториям.
